## 기능 요구 사항
- [X] 입력 기능
    - [X] 방문 날짜 입력
    - [X] 주문할 메뉴 및 개수 입력

- [X] 출력 기능
    - [X] 주문한 메뉴 리스트
    - [X] 할인 전 총 주문 금액
    - [X] 증정 이벤트 유무
    - [X] 혜택 내역 (적용된 할인들)
    - [X] 총혜택 금액
    - [X] 할인 후 예상 결제 금액
    - [X] 이벤트 배지

- [X] 계산 기능
    - [X] 총 주문 금액 계산
    - [X] 각종 할인 적용
    - [X] 증정 이벤트 적용
    - [X] 총혜택 금액 계산
    - [X] 할인 후 최종 결제 금액 계산
    - [X] 이벤트 배지 부여

- [X] 할인 종류
    - [X] 크리스마스 디데이 할인 (12월 1일 ~ 25일)
        - [X] 12월 1일: -1,000원
        - [X] 이후 매일 100원씩 증가 -> 12월 25일에는 -3,400원 할인
    - [X] 평일 할인 (일요일~목요일)
        - [X] 디저트 메뉴 1개당 -2,023원 할인
    - [X] 주말 할인 (금요일, 토요일)
        - [X] 메인 메뉴 1개당 -2,023원 할인
    - [X] 특별 할인 (달력에 ⭐이 있는 날)
        - [X] -1,000원 추가 할인
    - [X] 증정 이벤트
        - [X] 할인 전 총 주문 금액 12만 원 이상이면 샴페인(25,000원) 증정

- [X] 이벤트 배지 기준
    - [X] 총혜택 금액 5,000원 이상 -> 별
    - [X] 총혜택 금액 10,000원 이상 -> 트리
    - [X] 총혜택 금액 20,000원 이상 -> 산타
    - [X] 총혜택 금액 5,000원 미만 -> 없음

- [X] 예외 처리 기능
    - [X] 사용자가 잘못된 값을 입력할 경우 ValueError를 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 재입력 요청
    - [X] 방문 날짜 입력 예외 사항
        - [X] 방문 날짜가 1~31 사이의 숫자가 아닌 경우
    - [X] 주문할 메뉴 입력
        - [X] 잘못된 형식인 경우
        - [X] 메뉴판에 있는 항목이 아닌 경우
        - [X] 음료만 주문하는 경우
    - [X] 메뉴 개수 입력
        - [X] 잘못된 형식인 경우
        - [X] 메뉴 개수가 1 미만일 경우
        - [X] 메뉴 개수가 20개 초과일 경우

## 추가 기능
    - [X] InputView 클래스에서 사용자 입력 처리
    - [X] OutPutView 클래스에서 결과 출력
    - [X] DiscountCalculator 클래스를 만들어 할인 로직 관리
    - [X] EventPlanner 클래스를 만들어 전체적인 이벤트 적용 로직 관리

## 오류 수정
- [X] 할인 후 예상 결제 금액 출력 시, 증정 메뉴의 가격까지 포함되어 제외되는 오류 수정
